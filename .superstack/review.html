Architectural Risks (not code bugs)

  • POD-DEIM provenance. The whitepaper says training is offline; SOL pricer on mainnet uses solve_fair_coupon_e11_from_const — a const fn over a precompiled table. Correct in code. Ensure protocol_config.pod_deim_table_sha256 is verified against the training artefact before mainnet and committed in submission materials. A judge will ask.
  • Keeper pause. There is no global kill-switch across keeper-signed instructions. If the observation keeper is compromised, rotate_keeper closes the window but leaves a gap before admin notices. Consider an admin-settable keepers_paused flag in ProtocolConfig checked at the top of each keeper instruction.
  • Self-hosting during judging. Home NUC + home ISP + Colosseum review window = operational single point of failure. systemd hardening is great; network availability isn't something systemd solves. Strongly recommend Vercel/Cloudflare for the marketing site + keep keepers on the NUC.

What's clean (don't re-open)

  • Arithmetic: checked_add / checked_mul / checked_sub used consistently in value-sensitive paths. No stray + / * on money.
  • Signer/authority: has_one = admin on admin instructions; require_keys_eq!(keeper, registry.observation, ...) on keeper instructions — stricter than a bare Signer.
  • PDA bump canonicalization: bump stored in account state; seeds include unique identifiers (user pk, mint); discriminator-only cross-program-account pattern is the documented Anchor 0.32.1 workaround, defensible.
  • Off-chain secrets: .env not in git. .gitignore covers keepers/*/config/*.json. CI uses pinned action SHAs. Anchor.toml points at user keypair, not committed.
  • Frontend allowlist: runtime config locked to compile-time allowlist; genesis-hash cluster check prevents cluster spoofing from poisoned localStorage.
  • Keeper daemons: exponential backoff + failure budget + systemd PrivateTmp=true, ProtectSystem=strict, NoNewPrivileges=true, MemoryDenyWriteExecute=true. This is production-grade.
  • CI invariants: pure-Rust quote crates cannot transitively depend on anchor-lang / solana-program. Precision baseline regression guard. cargo audit with documented waivers.
